## 슈팅 게임 기획 가이드 (shot)

- 버전: `2026-07-02`
- 기준: `/arcade/` 하위 세로 스크롤 슈팅 게임 (단일 HTML)
- 상위 규칙: 본 문서는 [make-game.md](./make-game.md)를 상속한다. 공통 규칙(위치·메타·광고·레이아웃·등록·검수)은 그 문서를 따르고, 여기서는 슈팅 게임 고유 기획만 정의한다.

1. **개요**
   - 장르: 세로 스크롤 슈팅(shoot 'em up). 플레이어 기체가 화면 하단, 적이 상단에서 내려온다.
   - 목표: 적을 격추해 점수를 올리고, 웨이브를 넘기며 최대한 오래 생존한다.
   - 권장 경로: `/arcade/space-shooter.html`, 슬러그 `space-shooter`.
   - 톤: 우주/네온 다크 테마. 액센트 색은 시안/인디고 계열 권장.

2. **엔진/기술 규칙 (필수)**
   - 반드시 **Excalibur.js**로 제작한다. (액터/씬/게임루프/충돌 시스템 활용)
     - CDN: `https://cdn.jsdelivr.net/npm/excalibur@0.30.3/build/dist/excalibur.min.js` (전역 `ex`)
   - 캔버스 id는 `gameCanvas`, `displayMode: ex.DisplayMode.FitContainer`, `pointerScope: ex.PointerScope.Canvas`.
   - 내부 해상도 권장: `480 x 640`(세로), 컨테이너는 `aspect-[3/4]`.
   - 충돌은 Excalibur 충돌 그룹/이벤트를 사용한다.
     - 그룹: `player`, `playerBullet`, `enemy`, `enemyBullet`, `item`
     - 규칙: `playerBullet↔enemy`, `player↔enemy`, `player↔enemyBullet`, `player↔item`만 충돌하도록 collisionGroup을 구성한다. (같은 진영끼리는 충돌하지 않음)
   - 총알/적은 액터 풀(재사용) 또는 화면 밖 `kill()`로 정리한다.
   - 사이드바 "조작 방법" 하단에 `Powered by Excalibur.js`(https://excaliburjs.com/) 표기.

3. **플레이어 규칙**
   - 화면 하단 중앙에서 시작한다. 체력(HP) 또는 목숨(라이프) 중 하나로 생존을 관리한다. (권장: 라이프 3 + 피격 시 짧은 무적)
   - 이동: 포인터(마우스/터치)를 따라 기체가 부드럽게 이동. 화면 경계 안으로 clamp.
   - 사격: 기본 자동 연사(발사 간격 쿨다운). 스페이스/클릭 유지로도 발사 가능.
   - 피격: 적 또는 적 탄에 맞으면 라이프 감소 + 무적 깜빡임 + 화면 흔들림(`camera.shake`).
   - 파워업 단계: 무기 레벨(1→최대 3~4). 레벨업 시 탄 수/발사 패턴 강화(1발→2발→3발/확산).

4. **적 / 웨이브 규칙**
   - 최소 3종 이상의 적 타입을 정의한다. (예)
     - 잡몹: 직선 하강, 저체력, 미발사 또는 단발.
     - 사수: 지그재그/좌우 이동하며 플레이어 방향으로 탄 발사.
     - 돌격형: 빠르게 내려와 충돌 시도.
   - 스폰은 시간/웨이브 기반으로 점증한다. (스폰 간격 감소, 적 속도·체력·탄 수 증가)
   - 매 N 웨이브(권장 5)마다 **보스**를 등장시킨다.
     - 보스: 대형 스프라이트, 큰 체력바(HUD 상단), 다중 탄막 패턴, 격파 시 대량 점수/파워업 드롭.
   - 적/적탄이 화면 하단 밖으로 나가면 제거한다. (플레이어 미스로 카운트하지 않음)

5. **아이템 / 파워업 규칙**
   - 적 격파 시 일정 확률로 드롭한다. (아이템은 아래로 천천히 하강, 접촉 시 획득)
   - 최소 항목: 무기 강화(P), 라이프 회복(♥), 일시 보호막/봄(선택).
   - 봄(폭탄) 사용 시 화면 내 적 탄 제거 + 광역 대미지(선택 기능, 개수 제한).

6. **점수 / 난이도 / 기록 규칙**
   - 점수: 적 처치 점수 + 콤보/연속 격파 보너스 + 보스 보너스.
   - 콤보: 짧은 시간 내 연속 격추 시 배수 증가, 피격/시간 초과 시 초기화.
   - 난이도: 경과 시간·웨이브에 따라 스폰율/적 속도/탄 속도를 곡선으로 상승.
   - 최고 점수는 `localStorage`(예: `spaceShooterBest`)에 저장/복원한다.
   - 게임오버 시 오버레이에 최종 점수·웨이브·최고 점수를 표시하고 다시하기를 제공한다.

7. **조작 규칙**
   - 마우스: 커서를 따라 이동, 클릭 유지 시 발사(자동 연사 병행).
   - 터치: 드래그로 이동, 화면 터치 중 자동 발사.
   - 키보드(보조): 방향키/WASD 이동, 스페이스 발사, `B` 봄, `P`/`ESC` 일시정지.
   - 모바일에서 손가락에 기체가 가리지 않도록 약간의 오프셋을 둔다. (권장)

8. **UI / HUD / 화면 상태 규칙**
   - 상태: `idle`(시작 대기) → `playing` → `paused` → `over`.
   - HUD: 점수, 최고 점수, 웨이브, 라이프, 무기 레벨, (보스전) 보스 체력바.
   - 오버레이: 시작 안내, 일시정지, 게임오버 결과 화면.
   - 사이드바: 기록 카드 + 조작 방법(+엔진 표기). 하단 채팅/푸터는 공통 규칙 유지.

9. **연출 / 사운드 규칙**
   - 스크롤되는 별/성운 배경(패럴럭스 권장)으로 속도감을 준다.
   - 피격·격파 시 파편/폭발 파티클, 피격 시 화면 흔들림.
   - 사운드는 선택 사항이며, 넣을 경우 on/off 토글과 음소거 기본 대응을 제공한다.
   - 성능: 총알/적/파티클 수 상한을 두고 오프스크린 정리로 프레임을 유지한다.

10. **사이트 연동 / 등록 규칙**
   - [make-game.md](./make-game.md) 8항을 따른다.
     - `/arcade/index.html`에 새 타일(`.shot-tile` 그라데이션) 추가 + 소개문/메타 반영.
     - `sitemap.xml`에 `https://partygame.pe.kr/arcade/space-shooter.html` 추가.
     - 기존 게임 페이지 3-카드 상호 링크에도 노출.

11. **검수 체크리스트**
   - Excalibur CDN이 게임 스크립트보다 먼저 로드되는지 확인
   - 이동/자동연사, 3종 이상 적, 웨이브 점증, 보스 등장 확인
   - 충돌 그룹이 진영별로 올바른지 확인(아군탄이 적에게, 적탄이 아군에게만)
   - 파워업/라이프, 점수·콤보·최고 점수 저장, 게임오버·다시하기 확인
   - 마우스/터치/키보드 조작 확인, 모바일 레이아웃 확인
   - `Powered by Excalibur.js` 표기, `/arcade/` 타일 + `sitemap.xml` 등록 확인
   - SEO 태그(`title`, `description`, `canonical`, `og:*`) 확인
   - 배포 전: 인라인 스크립트 구문 검사 + (가능 시) 엔진으로 초기화/수 프레임 런타임 확인
