## 여행 RPG 시뮬레이션 기획 가이드 (simulation)

- 버전: `2026-07-06`
- 기준: `/arcade/` 하위 무한 랜덤 맵 여행 RPG 시뮬레이션 (단일 HTML)
- 상위 규칙: 본 문서는 [make-game.md](../make-game.md)를 상속한다. 공통 규칙(위치·메타·광고·레이아웃·등록·검수)은 그 문서를 따르고, 여기서는 여행 RPG 시뮬레이션 고유 기획만 정의한다.

1. **개요**
   - 장르: 무한 랜덤 맵을 여행하는 RPG 시뮬레이션. 한 소년이 끝없이 펼쳐지는 세계를 여행하며 몬스터를 물리치고 아이템을 모아 성장한다.
   - 목표: 최대한 멀리 여행하며 몬스터를 처치하고, 떨어진 아이템을 착용해 소년을 강하게 키운다. 소년이 쓰러지면(HP 0) 여정이 끝난다.
   - 핵심 루프: `전진(여행) → 몬스터 조우/전투 → 아이템 드롭/획득 → 착용·성장 → 더 강한 지역으로 전진`.
   - 진행 방식: 소년은 스스로 전진하는 **자동 여행(시뮬레이션)**을 기본으로 하고, 플레이어는 이동(회피)·전투·장비 관리에 개입한다. (완전 자동 방치형이 아니라 개입형 시뮬레이션)
   - 권장 경로: `/arcade/rpg-journey.html`, 슬러그 `rpg-journey`.
   - 톤: 따뜻한 판타지 모험 테마. 낮/노을/밤/설원 등 바이옴별 색을 달리하고, 액센트는 에메랄드/앰버 계열 권장.

2. **엔진/기술 규칙 (필수)**
   - 반드시 **Excalibur.js**로 제작한다. (액터/씬/게임루프/카메라/충돌 시스템 활용)
     - CDN: `https://cdn.jsdelivr.net/npm/excalibur@0.30.3/build/dist/excalibur.min.js` (전역 `ex`)
   - 캔버스 id는 `gameCanvas`, `displayMode: ex.DisplayMode.FitContainer`, `pointerScope: ex.PointerScope.Canvas`.
   - 내부 해상도 권장: 가로형 여행에 맞춰 `640 x 480`(가로) 또는 `480 x 640`(세로 스크롤 여행) 중 하나로 고정하고 컨테이너 `aspect`를 일치시킨다.
   - 게임 영역에 Fullscreen API 기반 전체화면 버튼을 제공한다. `F` 키로 진입/해제할 수 있어야 하며, 전체화면에서도 게임 비율을 유지하고 종료 버튼이 보여야 한다.
   - 충돌은 Excalibur 충돌 그룹/이벤트를 사용한다.
     - 그룹: `player`, `playerAttack`, `monster`, `monsterAttack`, `item`
     - 규칙: `playerAttack↔monster`, `player↔monster`, `player↔monsterAttack`, `player↔item`만 충돌하도록 collisionGroup을 구성한다. (같은 진영끼리는 충돌하지 않음)
   - 무한 맵/오브젝트는 화면 밖으로 벗어난 청크·몬스터·아이템·투사체를 `kill()`(또는 풀 재사용)로 정리해 메모리와 프레임을 유지한다.
   - 성능: 몬스터/아이템/투사체/파티클 수에 상한을 두고 오프스크린 정리로 저사양·모바일 프레임을 지킨다.
   - 사이드바 "조작 방법" 하단에 `Powered by Excalibur.js`(https://excaliburjs.com/) 표기.

3. **맵 / 월드 생성 규칙 (무한 랜덤)**
   - 맵은 **절차적(procedural)으로 무한 생성**한다. 여행이 진행되는 방향으로 새 지형을 계속 이어 붙이고, 지나온 지형은 정리한다.
   - 청크(구간) 단위로 생성한다. 각 청크는 지형 타일, 장식물(나무·바위·풀), 몬스터 스폰 지점, 아이템/보물 배치를 포함한다.
   - **시드 기반 랜덤**을 권장한다. 같은 시드는 같은 맵을 재현하고, 새 게임마다 시드를 바꿔 매번 다른 여정을 만든다.
   - 바이옴(지역)을 최소 3종 이상 정의한다. (예: 초원 → 숲 → 사막/설원 → 동굴/화산) 여행 거리가 늘수록 더 위험한 바이옴으로 전환하고 배경·몬스터·드롭이 달라진다.
   - 여행 거리(경과 이동량)를 누적해 난이도·바이옴·스폰율의 기준으로 삼는다.
   - 배경은 패럴럭스 다층 스크롤로 이동감을 준다. (원경/중경/근경)

4. **플레이어(소년) 규칙**
   - 주인공은 여행하는 소년이다. 이동(걷기/달리기) 애니메이션 또는 최소한의 상태 표현(걷기·피격·공격)을 준다.
   - 기본 스탯을 정의한다: `HP`, `공격력(ATK)`, `방어력(DEF)`, `이동속도(SPD)`, `치명타/공격속도(선택)`, `경험치(EXP)`, `레벨(LV)`, `골드(선택)`.
   - **성장(레벨업)**: 몬스터 처치로 EXP를 얻고 레벨업 시 스탯이 상승한다. 레벨업 연출(플래시/파티클/HP 회복 등)을 준다.
   - **장비 착용으로 성장**: 아이템을 착용하면 외형과 능력치에 반영되어야 한다.
     - 장비 슬롯 최소 3종: `무기`, `방어구(갑옷)`, `장신구(악세서리)`. 각 슬롯은 하나만 착용하고 더 좋은 것으로 교체한다.
     - 착용 시 캐릭터 스프라이트에 무기/방어구가 눈에 보이게 반영되면 가산점. (최소한 색·아이콘 변화)
   - 이동: 포인터(마우스/터치)·키보드로 소년을 좌우(또는 상하)로 움직여 몬스터·함정을 피하고 아이템을 줍는다. 화면 경계 안으로 clamp.
   - 전투: 기본 자동 공격(전방/근처 몬스터 대상, 쿨다운). 무기 종류에 따라 근접 베기·원거리 투사체·범위 공격 등으로 구분한다.
   - 피격: 몬스터 접촉/몬스터 공격에 맞으면 HP 감소 + 짧은 무적 깜빡임 + 화면 흔들림(`camera.shake`). HP가 0이 되면 게임오버.

5. **몬스터 / 전투 규칙**
   - 몬스터는 맵 진행에 따라 **랜덤으로 등장**한다. 최소 3종 이상 정의한다. (예)
     - 잡몹: 느리게 접근, 저체력, 근접.
     - 원거리형: 멈춰서 소년에게 투사체 발사.
     - 돌진형: 빠르게 달려와 충돌 시도.
   - 스폰은 여행 거리·시간 기반으로 점증한다. (스폰 간격 감소, 몬스터 속도·체력·공격력 증가, 동시 출현 수 증가)
   - 바이옴이 바뀌면 그 지역 고유 몬스터가 등장하고 외형·패턴이 달라진다.
   - 일정 거리마다(권장 일정 구간) **보스/정예 몬스터**를 등장시킨다.
     - 대형 외형, 체력바(HUD), 다중 공격 패턴, 처치 시 대량 EXP·희귀 아이템 드롭.
     - 여행이 깊어질수록 보스 위협 단계를 올리고 외형·패턴이 강화되어야 한다.
   - 몬스터가 화면 밖(뒤쪽)으로 완전히 벗어나면 정리한다. (미처치는 미스로 카운트하지 않되, 접촉 위험은 유지)

6. **아이템 / 장비 / 성장 규칙**
   - 몬스터 처치·보물 상자·맵 배치로 아이템이 **드롭**된다. 드롭 아이템은 잠깐 표시되며 소년이 접촉하면 획득한다.
   - 아이템 등급(레어리티)을 최소 3단계 둔다. (예: 일반/희귀/전설) 등급이 높을수록 능력치 보너스가 크고 색·연출로 구분한다.
   - 아이템 분류(최소):
     - 장비(무기/방어구/장신구): 착용 시 능력치 상승, 슬롯당 교체.
     - 소비: 포션(HP 회복), 버프(일시 공격/속도 증가) 등.
     - 재화(선택): 골드/보석으로 상점·강화에 사용.
   - 착용 규칙: 더 좋은 장비를 주우면 교체 여부를 처리한다. (자동 착용 또는 간단한 비교 UI) 교체 시 스탯 변화가 HUD에 즉시 반영되어야 한다.
   - 인벤토리/장비 상태를 HUD 또는 패널에 표시한다. (현재 무기/방어구/장신구 + 주요 스탯)

7. **진행 / 난이도 / 기록 규칙**
   - 점수: 여행 거리 + 몬스터 처치 + 보스 보너스 + 획득 아이템/골드 기반으로 산정한다.
   - 난이도: 여행 거리·경과 시간에 따라 스폰율/몬스터 능력치/바이옴 위험도를 곡선으로 상승시킨다.
   - 최고 기록은 `localStorage`(예: `rpgJourneyBest`)에 저장/복원한다. (최고 거리 또는 최고 점수)
   - 게임오버 시 오버레이에 최종 거리·레벨·점수·최고 기록을 표시하고 다시하기(새 시드 여행)를 제공한다.

8. **조작 규칙**
   - 마우스: 커서/드래그로 소년 이동, 자동 전투 병행. 클릭으로 상호작용(획득/교체·스킬 선택 등, 게임에 맞게).
   - 터치: 드래그로 이동, 이동 중 자동 전투. 모바일에서 손가락에 소년이 가리지 않도록 약간의 오프셋을 둔다.
   - 키보드(보조): 방향키/WASD 이동, 스페이스/클릭 공격 또는 상호작용, `P` 일시정지, `F` 전체화면 진입/해제. (`ESC`는 브라우저 전체화면 해제와 충돌하므로 전체화면 종료를 우선한다.)

9. **UI / HUD / 화면 상태 규칙**
   - 상태: `idle`(시작 대기) → `playing` → `paused` → `over`.
   - HUD: HP(막대), 레벨 + 경험치바, 공격/방어 등 주요 스탯, 착용 장비(무기/방어구/장신구), 여행 거리, 골드, (보스전) 보스 체력바.
   - 오버레이: 시작 안내, 일시정지, 게임오버 결과 화면. 시작 전 소년 이름/외형/시작 특성 선택을 제공하면 가산점.
   - 사이드바: 기록 카드(거리·레벨·최고) + 조작 방법(+엔진 표기). 하단 채팅/푸터는 공통 규칙 유지.

10. **연출 / 사운드 규칙**
   - 바이옴별 배경·색조 변화, 패럴럭스 스크롤로 여행감을 준다.
   - 타격·처치·레벨업·아이템 획득에 파티클/플래시, 피격 시 화면 흔들림.
   - 희귀 이상 아이템 드롭 시 강조 연출(빛기둥/색상)로 구분한다.
   - 사운드는 선택 사항이며, 넣을 경우 on/off 토글과 음소거 기본 대응을 제공한다.

11. **사이트 연동 / 등록 규칙**
   - [make-game.md](../make-game.md) 8항을 따른다.
     - `/arcade/index.html`에 새 타일(`.rpg-tile` 그라데이션) 추가 + 소개문/메타 반영.
     - `sitemap.xml`에 `https://partygame.pe.kr/arcade/rpg-journey.html` 추가.
     - 기존 게임 페이지 3-카드 상호 링크에도 노출.

12. **검수 체크리스트**
   - Excalibur CDN이 게임 스크립트보다 먼저 로드되는지 확인
   - 맵이 무한·랜덤(시드 기반)으로 이어지고 오프스크린 청크가 정리되는지 확인
   - 바이옴 3종 이상 전환과 거리 기반 난이도 상승 확인
   - 소년 이동/자동 전투, 레벨업 성장, HP·EXP·스탯 HUD 반영 확인
   - 무기/방어구/장신구 3슬롯 착용·교체와 능력치·외형 반영 확인
   - 몬스터 3종 이상, 랜덤 스폰 점증, 보스/정예 등장과 강화 확인
   - 아이템 드롭·획득, 등급(레어리티) 구분, 소비 아이템(포션 등) 확인
   - 충돌 그룹이 진영별로 올바른지 확인(아군 공격이 몬스터에게, 몬스터 공격이 소년에게만)
   - 점수/거리 집계, 최고 기록 저장, 게임오버·다시하기(새 시드) 확인
   - 마우스/터치/키보드 조작 확인, 모바일 레이아웃·터치 하드닝 확인
   - 전체화면 버튼/`F` 키로 진입·해제되고 게임 비율과 종료 버튼이 유지되는지 확인
   - `Powered by Excalibur.js` 표기, `/arcade/` 타일 + `sitemap.xml` 등록 확인
   - SEO 태그(`title`, `description`, `canonical`, `og:*`) 확인
   - 배포 전: 인라인 스크립트 구문 검사 + (가능 시) 엔진으로 초기화/수 프레임 런타임 확인
